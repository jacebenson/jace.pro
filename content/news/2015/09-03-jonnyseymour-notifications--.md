<p>In <a title="" _jive_internal="true" data-containerid="-1" data-containertype="-1" data-objectid="45437" data-objecttype="3" href="/people/jonnyseymour/content"> my past blog posts</a> I have talked about <a title="" _jive_internal="true" href="/wiki.servicenow.com/index.php?title=Email_Notifications" rel="nofollow" target="_blank">notifications</a>. I will discuss now how the notification option "weight" (and not the one you have to be on a diet) affects the outgoing notifications. Feel free to comment.</p><p><br/>Use <a title="" _jive_internal="true" href="/wiki.servicenow.com/index.php?title=Email_Notifications#gsc.tab=0" rel="nofollow" target="_blank">email notifications</a> to send selected users email about specific activities in the system, such as updates to incidents or change requests. One of the options is the 'weight'. The email notification with the highest weight is sent but only if the generated emails that are being compared have the same recipients and target table.</p><p></p><p><img   alt="winner1.png" class="image-1 jive-image" height="117" src="8d9bff7ddb901304b322f4621f961901.iix" style="width: 204px; height: 116.806px; display: block; margin-left: auto; margin-right: auto;" width="204"/></p><p></p><h2>Email notifications weight</h2><p></p><p>Email notification weight is defined as a required numerical value for this notification's priority relative to other notifications:</p><ul style="list-style-type: disc;"><li><span style="color: blue;"><strong>With the same target table and recipients</strong></span></li><li><strong>The system only sends the notification with the highest weight </strong></li></ul><p></p><p>All other notifications with lowest weight are moved from the Outbox mailbox to the Skipped mailbox. So far so good.</p><p>There is a 'Order' setting on the Email notification.   On my testing, 'Order' does not affect the behavior.</p><p>The default value weight 0 causes the system to always send the notification (assuming the conditions are met).</p><p></p><p></p><p></p><p><img   alt="2015-09-02_1411.png" class="image-0 jive-image" src="ec7dbb3ddb945fc03eb27a9e0f961936.iix" style="height: 144px; width: 620px;"/></p><p></p><p>Email notifications have great features that allow you to add priority to certain notifications over others. However, weight can be misunderstood.</p><pre __default_attr="warning" __jive_macro_name="alert" alert="warning" class="jive_text_macro jive_macro_alert" data-renderedposition="810.2000122070312_8_917_43"><p>Weight will not avoid duplicate notification ON MANY CASES as weight is considered based on the recipient list and target table.</p></pre><h2>How email notification "weight" option works</h2><p></p><p>When the system is sending a notification:</p><ol><li>If an event is triggered (e.g. Issue is commented), the email notification matching the conditions will execute.</li><li>Disregarding weight, if the notification condition passes and it generates valid recipients (emails), it generates an outbound email.</li><li>If the outbound emails <strong>HAVE EXACTLY SAME 'Recipient' and 'Target'</strong>, only the one with highest weight is sent plus any one with weight=0. <br/> The rest of the emails are marked as <strong>'State'="Ignored"</strong></li></ol><p></p><p><span style="background: yellow;"><img   alt="2015-09-02_1442.png" class="image-0 jive-image" src="332cf406db501b04ed6af3231f9619bb.iix" style="height: 136px; width: 620px;"/><br/></span></p><p><span style="background: yellow;"><img   alt="email weight.jpeg" class="image-0 jive-image" src="3c0ed446db1057041dcaf3231f9619ee.iix" style="height: 308px; width: 620px;"/><br/></span></p><pre __default_attr="warning" __jive_macro_name="alert" alert="warning" class="jive_text_macro jive_macro_alert" data-renderedposition="1561.4000244140625_8_917_43"><p>On the email, the target and the recipients need to match so the weight is compared.</p></pre><p></p><h2>Testing how email notification "weight" option works</h2><p></p><p>Let me run some examples:</p><p></p><p>Assuming the following notifications:</p><table border="1" cellpadding="0" cellspacing="0" style="table-layout: fixed;
